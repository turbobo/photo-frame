# Photo Frame · 相框水印工具 — 技术方案文档

> 版本：v1.0.0 · 最后更新：2026-07-02

---

## 一、项目定位

Photo Frame 是一款**纯浏览器端**的相机照片边框水印工具，为国内用户提供以下能力：

- 快速为照片添加**摄影师风格边框**（极简/拍立得/胶片/EXIF 参数栏/社交卡片）
- 自动读取 EXIF 元数据（相机型号、镜头、光圈、快门、ISO、日期、GPS）
- 自动匹配 14 个主流相机品牌 Logo（Sony/Canon/Nikon/Fujifilm/Leica...）
- 支持 **RAW / JPG / PNG / HEIC** 全部主流拍摄格式
- 一键导出多种尺寸和格式

**目标用户**：摄影爱好者、旅行博主、社交媒体分享（小红书/微博/朋友圈）

## 二、技术选型

| 维度 | 选型 | 版本 | 理由 |
|------|------|------|------|
| 构建工具 | **Vite** | 5.4 | 冷启动毫秒级、HMR 极快、生态完整 |
| 前端框架 | **React** | 18.3 | 组件生态、Hook API 心智负担低 |
| 类型系统 | **TypeScript** | 5.4 | 严格模式、Vite 原生支持 |
| 样式方案 | **Tailwind CSS** | 3.4 | 原子化、体积小、暗色主题友好 |
| EXIF 解析 | **exifr** | 7.1 | 支持 RAW 内嵌预览、GPS、XMP、覆盖 20+ 格式 |
| 图像处理 | **Canvas 2D API** | 浏览器原生 | 零依赖、DPR 缩放、导出 blob |
| 部署平台 | **EdgeOne Pages** | 免费版 | 国内 CDN 加速、静态托管、免费 10GB/月 |

**为什么不选 Next.js + Sharp**（对照方案 B）：
- EdgeOne Pages 免费版**无 Node 运行时**，Sharp 无法运行
- Sharp 的 RAW 支持需要 libraw 编译，二进制 100MB+，超过 Serverless 免费额度
- 图片处理放浏览器 = **零服务器带宽 + 零上传等待 + 用户隐私更好**

## 三、格式支持策略

### 3.1 支持清单

| 类别 | 格式 | 处理方式 |
|------|------|---------|
| 常规位图 | JPG/JPEG · PNG · WEBP · AVIF | Canvas 原生 `Image` 加载 |
| 移动端 | HEIC/HEIF（iPhone 拍摄） | exifr 提取内嵌 JPEG 预览 |
| **RAW** | CR2/CR3（Canon）· NEF（Nikon）· ARW/SR2（Sony）· RAF（Fujifilm）· RW2（Panasonic）· ORF（Olympus）· PEF（Pentax）· DNG（通用）· RWL（Leica） | exifr 提取内嵌 full-resolution JPEG |

### 3.2 RAW 处理原理

RAW 文件内**都嵌有一张 full-resolution JPEG 预览**（相机屏幕回放用）。exifr 直接读取这张 JPEG 用于渲染，不需要解 Bayer 传感器数据（那需要 dcraw/libraw，无法在纯浏览器实现）。

**优势**：
- 秒开（提取只需读 IFD 段，< 500ms）
- 无需服务器
- 相机厂商已经调过色，视觉效果良好

**局限**：
- 不能做 RAW 显影（曝光、白平衡等参数）
- 预览分辨率一般接近原图但不完全等同

### 3.3 关键代码

`src/utils/exif.ts::loadImage()`：

```ts
export async function loadImage(file: File): Promise<HTMLImageElement> {
  if (isNativeImage(file)) return loadImageFromBlob(file)
  const { thumbnail } = await import('exifr')     // 按需加载
  const buf = await thumbnail(file)                // 提取内嵌 JPEG
  const blob = new Blob([buf], { type: 'image/jpeg' })
  return loadImageFromBlob(blob)
}
```

### 4.2 渲染统一入口

`src/utils/canvas.ts`：

```ts
export function renderFrame(ctx: RenderCtx): HTMLCanvasElement {
  switch (ctx.config.id) {
    case 'minimal':  return renderMinimal(ctx)
    case 'polaroid': return renderPolaroid(ctx)
    case 'film':     return renderFilm(ctx)
    case 'exif':     return renderExif(ctx)
    case 'insta':    return renderInsta(ctx)
    case 'leica':    return renderLeica(ctx)
    case 'red-dot':  return renderRedDot(ctx)
    case 'dazz':     return renderDazz(ctx)
    case 'instax':   return renderInstax(ctx)
    case 'xhs':      return renderXhs(ctx)
    case 'vintage':  return renderVintage(ctx)
    case 'magazine': return renderMagazine(ctx)
    case 'location': return renderLocation(ctx)
  }
}
```

每个渲染器接收：
- `image: HTMLImageElement` — 原图
- `exif: ExifData` — 元数据
- `logo: HTMLImageElement | null` — 品牌 Logo
- `config: TemplateConfig` — 用户参数（padding/color/fontSize...）

`TemplateConfig` 字段（含新增 `locationName`）：

```ts
interface TemplateConfig {
  id: TemplateId           // 13 种模板之一
  padding: number          // 边距 0-30 (%)
  bgColor: string          // 边框颜色
  textColor: string        // 文字颜色
  showLogo: boolean        // 显示品牌 Logo
  showExif: boolean        // 显示 EXIF 信息
  customText: string       // 自定义文字（覆盖默认）
  fontSize: number         // 字号相对图片长边百分比 1-5
  logoSize: number         // Logo 相对图片长边百分比 2-10
  radius: number           // 圆角 0-80 px
  shadow: boolean          // 卡片阴影
  locationName?: string    // 地点名（location 模板专用）
}
```

返回：一个 `HTMLCanvasElement`，用于预览与导出。

### 4.3 尺寸与字号策略

所有尺寸参数使用**图片长边百分比**表达，确保不同分辨率下视觉一致：

```ts
const long = Math.max(W, H)
const pad  = long * config.padding / 100      // 边距 5% → 20MP 图片是 200px
const font = long * config.fontSize / 100     // 字号 2%  → 约 100px
```

### 5.3 交互特性

- **拖拽**：整个上传区支持 dragover/drop，视觉反馈 dropzone-active
- **粘贴**：全局监听 Cmd+V，从剪贴板获取图片
- **实时预览**：config 每次变化 → useEffect 重绘 canvas
- **DPR 适配**：预览 canvas 使用 devicePixelRatio 高清渲染
- **自适应缩放**：ResizeObserver 监控容器变化，长边自动限制在可视区

## 六、数据流

```
User 上传 File
    ↓
loadImage(file)   ─┐
extractExif(file) ─┼→ 并行 (Promise.all)
    ↓              │
    ├─ HTMLImageElement (原图/RAW 预览)
    └─ ExifData → getLogoPath(make) → loadLogo()
    ↓
setPhoto({ file, image, exif })  →  App state
    ↓
config 变化时 → renderFrame({image, exif, logo, config})
    ↓
返回 HTMLCanvasElement → 传给 PreviewCanvas 用 drawImage 缩放显示
    ↓
导出：re-renderFrame → toBlob(mime, quality) → download
```

## 七、性能预算

| 指标 | 实测 | 目标 |
|------|------|------|
| 首屏 JS gzip | 55 KB (index) | < 200 KB ✅ |
| exifr gzip | 26 KB (按需加载) | 首屏不加载 ✅ |
| CSS gzip | 4 KB | < 15 KB ✅ |
| 冷启动 (dev) | 380ms | < 1s ✅ |
| RAW 预览提取 | ~300ms（20MP 相机） | < 500ms ✅ |
| Canvas 重绘 | ~40ms（20MP 图 + 5 模板） | < 50ms ✅ |
| 生产构建 | 546ms | < 3s ✅ |

**优化措施**：
- exifr 按需 dynamic import（首屏不加载）
- Logo 使用 SVG（矢量、体积小、缓存友好）
- Canvas 一次性渲染，预览用 drawImage 二次缩放（避免重复计算）
- Vite 生产模式自动 tree-shaking + terser 压缩

## 十一、已知限制

| 限制 | 原因 | 缓解 |
|------|------|------|
| 大于 100MB 的 RAW 文件浏览器可能崩溃 | 内存限制 | 提示用户先降采样 |
| Safari 不支持 dynamic import 部分场景 | 兼容性 | 已用 Vite polyfill |
| 手写风字体 fallback | 系统字体差异 | 使用 Snell Roundhand → STXingkai 链 |
| Logo 文字风格非官方 | 商标合规 | 免费使用、可用户自定义 |
